# 文獻探討與解析 v1.0

## 核心哲學
「**好的文獻探討，不是文章的堆砌，而是知識圖的建構。**
你的任務是找出前人研究的脈絡，然後找到你能站上去的肩膀。」

## Step 2：研究情境確認

```
Q1：「你的研究主題是什麼？（一句話描述）」
Q2：「這是什麼類型的文獻工作？
    A. 論文第二章（文獻探討章節）
    B. 投稿期刊的 Related Work 章節
    C. 研究計畫書的文獻回顧
    D. 系統性文獻回顧（SLR/Scoping Review）
    E. 快速文獻梳理（了解現況）」
Q3：「你目前有哪些關鍵詞？
    （我會幫你優化搜尋策略）」
Q4：「你的研究領域的主要語言是中文還是英文？
    （影響資料庫選擇）」
Q5：「你希望覆蓋的年份範圍？
    例：2015-2025（近 10 年）」
```

## Step 3：搜尋策略設計

### 布林運算符邏輯
```
你的主題：[Topic]

主要關鍵詞組：
  中文：[詞1] + [詞2] + [詞3]
  英文：[Word1] AND [Word2]

替代詞彙（Synonyms）：
  [詞1] = [同義詞1] OR [同義詞2]

排除詞（避免不相關）：
  NOT [排除詞1]

最終搜尋字串示例：
  ("keyword1" OR "synonym1") AND "keyword2" 
  AND ("keyword3" OR "synonym3")
  NOT "exclusion_term"
```

### 資料庫搜尋清單
```
依優先序搜尋：

學術全文資料庫：
1. Google Scholar → 廣泛收錄（起點）
2. Web of Science → 高影響力期刊
3. Scopus → 跨學科覆蓋廣
4. ERIC（教育類）/ PsycINFO（心理類）

中文資料庫：
5. 台灣博碩士論文系統（ndltd.ncl.edu.tw）
6. HyRead 電子書（台灣中文期刊）
7. CNKI（中國知識基礎設施）
8. 臺灣期刊論文索引系統

語料庫/語言學（若適用）：
9. COCT / BNC / COCA
10. Linguistic Data Consortium
```

## Step 4：文獻主題聚類分析

收集到文獻清單後，執行聚類：

```
文獻分類框架：

群組 A：[主題 A 名稱]（[N] 篇）
  代表文獻：[Author, Year]; [Author, Year]
  核心觀點：___
  研究方法：___

群組 B：[主題 B 名稱]（[N] 篇）
  ...

研究趨勢分析：
  ・[年份前] 主要聚焦在…
  ・[年份後] 趨勢轉向…
  ・近期（2022-2025）熱點是…

研究方法分布：
  量化研究：__% 
  質性研究：__%
  混合研究：__%
```

---

## Step 5：研究缺口識別

```
根據文獻聚類，識別以下類型的缺口：

1. 主題缺口（Topic Gap）：
   「大多數研究探討 A，但幾乎沒有研究關注 B」

2. 方法論缺口（Methodological Gap）：
   「多為質性研究，缺乏量化驗證」

3. 情境缺口（Context Gap）：
   「主要在西方情境，台灣本土研究稀少」

4. 時間缺口（Temporal Gap）：
   「多為 2015 年前的研究，近年數位化影響尚未探討」

5. 對象缺口（Population Gap）：
   「多研究大學生，K-12 族群幾乎無研究」

你的研究定位：
「本研究填補了___缺口，
 透過___方法，在___情境中研究___族群。」
```

## Step 6：文獻綜述草稿生成

```
輸出格式（論文第二章段落結構）：

## 文獻探討

### 2.1 [主題 A]

[開頭段：概括此主題的研究重要性]

[文獻群組 A 的核心論述，含引用]
（Author, Year）指出___；
（Author & Author, Year）進一步發現___；
然而，（Author, Year）的研究卻發現不同結果，認為___。

[段落結語：引出你的研究問題]
```
