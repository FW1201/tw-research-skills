# 雙菱形學術研究計畫引導系統

## 概覽

本 Skill 將雙菱形設計思考（Double Diamond）框架對應到學術研究計畫發展的四個階段：

```
◇ 探索（Discover）→ ◆ 聚焦（Define）→ ◇ 發展（Develop）→ ◆ 交付（Deliver）
        問題域                    問題定義                方法域                計畫書
```

每個階段都結合：
1. 🔍 **Web Search** 查閱現有文獻與研究趨勢
2. 💬 **概念對齊對話** 與使用者確認方向
3. 📝 **結構化輸出** 填入計畫書模板

---

## 執行流程

### 準備工作（開始前必讀）

從 `references/proposal-template.md` 載入計畫書 MD 模板。
從 `references/diamond-phases.md` 載入各階段提問清單與引導語。

---

### 第一鑽（問題域 Problem Space）

#### Phase 1：Discover 廣泛探索（發散）

**目標**：協助使用者理解研究領域的全貌，避免過早收斂。

**步驟**：

1. 詢問初始興趣領域（1-2 個關鍵詞即可）
2. 立刻使用 `web_search` 搜尋：
   - 近 5 年相關領域的熱門議題
   - 常見的研究缺口（research gap）
   - 台灣學術脈絡中的相關研究
3. 將搜尋結果整理成「研究機會圖譜」呈現給使用者
4. 引導使用者用 HMW（How Might We）問句發散想法

**概念對齊問題（每個都問，等待使用者回答後再繼續）**：
- 「您對這個領域的哪個面向最感好奇或最有疑問？」
- 「您目前觀察到什麼現象讓您想研究這個主題？」
- 「這個研究如果成功，對誰最有幫助？」

**輸出**：一份「探索地圖」（含 3-5 個可能研究方向）

---

#### Phase 2：Define 問題聚焦（收斂）

**目標**：從廣泛探索中收斂出一個精準、可執行的研究問題。

**步驟**：

1. 回顧 Phase 1 的探索地圖
2. 再次使用 `web_search` 針對初步選定方向深入搜尋：
   - 現有研究是否已解決此問題？
   - 研究缺口是否真實存在？
   - 相關研究方法有哪些？
3. 與使用者進行「KAP 概念對齊檢查點」

**KAP 概念對齊問題**（必問，逐一確認）：
- 「這個研究問題，您希望是『描述性』（現象是什麼）、『解釋性』（為什麼）、還是『介入性』（怎麼做更好）？」
- 「您的研究對象（participants/context）是誰？有什麼條件限制？」
- 「這個問題在學術上的貢獻點是什麼？（理論貢獻 vs. 實務貢獻）」
- 「以您目前的時間與資源，這個研究規模可行嗎？」

**輸出**：確認版「研究問題陳述」（Research Problem Statement）

---

### 第二鑽（方法域 Solution Space）

#### Phase 3：Develop 方法發展（發散）

**目標**：為確定的研究問題探索多種可能的研究設計方案。

**步驟**：

1. 根據 Phase 2 的研究問題，使用 `web_search` 搜尋：
   - 同類型研究常用的研究設計
   - 量化 / 質性 / 混合方法的適配性
   - 資料收集工具範例（問卷、訪談、實驗等）
2. 提出 2-3 種研究設計選項供使用者評估
3. 引導使用者比較各方案的優缺點

**概念對齊問題**：
- 「您比較傾向量化（數字/統計）、質性（訪談/觀察）還是混合方法？」
- 「您能接觸到研究對象嗎？有無倫理審查限制？」
- 「您預計的研究時程是多久？」

**輸出**：「研究設計比較表」

---

#### Phase 4：Deliver 計畫書交付（收斂）

**目標**：整合所有對話內容，生成正式學術計畫書。

**步驟**：

1. 與使用者確認所有資訊完整後，宣告進入寫作階段
2. 依照 `references/proposal-template.md` 的結構逐章填入
3. 使用 `web_search` 補充具體引用文獻（至少 5 筆）
4. 生成完整 Markdown 草稿
5. 請使用者確認後，呼叫 docx skill 輸出 .docx 檔案

**輸出文件格式**：參見 `references/proposal-template.md`

## 跨階段規則

### Web Search 使用規則

```
Phase 1 搜尋：領域趨勢、研究機會（廣泛）
  → 搜尋語：「[領域] 研究趨勢 近年」「[領域] research gap」

Phase 2 搜尋：缺口驗證、方法初探（精準）
  → 搜尋語：「[具體問題] 相關研究 台灣」「[具體問題] systematic review」

Phase 3 搜尋：方法文獻、工具範例（實用）
  → 搜尋語：「[研究設計] 研究方法」「[量表/工具] reliability validity」

Phase 4 搜尋：關鍵文獻補充（引用用）
  → 搜尋語：具體作者、年份、期刊名稱驗證
```

### 概念對齊（KAP）原則

- **永遠等待使用者回答**，不要自行假設
- 每個 Phase 至少有一個確認問題得到肯定回答才能推進
- 若使用者回答模糊，追問一次再繼續
- 若使用者想跳過某 Phase，允許但提醒風險

### 語言規範

- 預設使用**繁體中文**
- 學術術語附英文對照（如：研究問題 / Research Question）
- 計畫書本文使用學術語體

## 呼叫 docx skill 輸出計畫書

當 Phase 4 完成後，使用以下流程：

1. 讀取 `references/docx-output-guide.md` 了解格式規格
2. 安裝 docx npm 套件
3. 用 Node.js 腳本生成 .docx，套用台灣學術格式：
   - 字型：標楷體 12pt（中文）、Times New Roman 12pt（英文）
   - 行距：1.5 倍
   - 頁邊距：上 3cm、下 2cm、左右各 3cm
   - 頁碼：阿拉伯數字，頁面下方置中
4. 輸出到 `/mnt/user-data/outputs/research_proposal.docx`
5. 使用 `present_files` 工具提供下載
